## Body

<ParamField body="model" type="string" required>
  Model name

  Fixed value: `"gpt_image_2"`
</ParamField>

<ParamField body="params" type="object" required>
  Model parameter object

  <Expandable title="params object properties">
    <ParamField body="prompt" type="string" required>
      Text description for image generation

      * Supports Chinese and English; detailed descriptions are recommended
      * Before submission, it will go through platform sensitive words / safety review; hitting prohibited content will return an error immediately
    </ParamField>

    <ParamField body="image_url" type="string[]">
      Array of reference images; providing this triggers image-to-image mode

      * Up to 10 reference images
      * Supports the following two formats:
        1. Publicly accessible URLs
        2. Base64 encoded format
           * Must use full Data URI format: `data:image/{format};base64,{base64 data}`
           * Supported image formats: jpeg, png, webp
           * Example: `data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABg...`
           * ⚠️ Note: Must include the prefix `data:image/jpeg;base64,`
    </ParamField>

    <ParamField body="aspect_ratio" type="string" default="auto">
      Aspect ratio for image generation

      Default value varies depending on `resolution`:

      * When `resolution` is `1K` / `2K`, not passing this defaults to `auto`
      * When `resolution` is `4K`, not passing this defaults to `16:9`

      Supported values:

      * `auto` - Automatically select the most suitable ratio
      * `1:1` - Square
      * `16:9` / `9:16` - Wide horizontal / vertical
      * `4:3` / `3:4` - Standard screen horizontal / vertical
      * `3:2` / `2:3` - Classic horizontal / vertical
      * `5:4` / `4:5` - Almost square horizontal / vertical
      * `2:1` / `1:2` - Wide horizontal / tall vertical
      * `21:9` / `9:21` - Ultra-wide horizontal / ultra-tall vertical

      <Warning>
        Only ratio strings or `auto` are supported; passing pixel dimensions (e.g. `1024x1024`) will cause an error
      </Warning>
    </ParamField>

    <ParamField body="resolution" type="string" default="1K">
      Resolution level

      Controls actual output image clarity:

      * `1K` - 1024 baseline, cost-effective for daily use (default)
      * `2K` - 2048 baseline, suitable for posters / HD needs
      * `4K` - 3840 baseline, **only supports 6 aspect ratios** (`16:9` / `9:16` / `2:1` / `1:2` / `21:9` / `9:21`)

      <Warning>
        Unsupported 4K combinations return 400:

        * `1:1` × `4K` ❌ (exceeds pixel limit)
        * `3:2` / `2:3` × `4K` ❌ (exceeds pixel limit)
        * `4:3` / `3:4` × `4K` ❌ (exceeds pixel limit)
        * `5:4` / `4:5` × `4K` ❌ (exceeds pixel limit)

        For HD images with these ratios, use `resolution=2K`.
      </Warning>

      <Warning>
        When passing `size`, `resolution` and `aspect_ratio` must not be passed simultaneously; otherwise, 400 is returned.
      </Warning>
    </ParamField>

    <ParamField body="size" type="string">
      Custom pixel size (**mutually exclusive** with `aspect_ratio` / `resolution`)

      Format is `widthxheight`, e.g. `2048x1536`, `3840x2160`.

      **Usage restrictions:**

      * Format must match `^\d+x\d+$`, such as `2048x1536`, other formats are unsupported
      * Mutually exclusive with `aspect_ratio` and `resolution`; only one of these three can be used at a time; passing together returns 400
      * Total pixels (width × height) must be within **655,360 ~ 8,294,400**

      **Measured pixel lower limit:**

      | Total Pixel Range | Upstream Behavior |
      | --- | --- |
      | ≥ 2,088,960 (approx 2.0M) | Output images precisely match passed `size` ✅ |
      | < 2,088,960 | Silently replaced with a close ratio of ~1.57M pixels (equivalent to not passing `size`) ⚠️ |

      <Warning>
        When total pixels are below approx 2,088,960, upstream does not report an error but silently replaces output size. For precise control, **total pixels ≥ 2,088,960** is recommended.

        For smaller sizes, use `aspect_ratio` + `resolution=1K` combination.
      </Warning>

      **Recommendations:**

      | Requirement | Recommended Method |
      | --- | --- |
      | Precise pixel size control | `size`, with total pixels ≥ 2,088,960 |
      | Small size (< 2M pixels) | `aspect_ratio` + `resolution=1K` |
      | Medium to large size any ratio | `size`, precise output supported in 2K / 4K ranges |
    </ParamField>
  </Expandable>
</ParamField>

## Size × Resolution Mapping Table

`aspect_ratio × resolution` → Actual pixels (13 ratios × 3 levels):

| aspect\_ratio | `1K`      | `2K`      | `4K`          |
| ------------- | --------- | --------- | ------------- |
| `1:1`         | 1024×1024 | 2048×2048 | ❌ Exceeds pixel limit       |
| `3:2`         | 1536×1024 | 2048×1360 | ❌ Exceeds pixel limit       |
| `2:3`         | 1024×1536 | 1360×2048 | ❌ Exceeds pixel limit       |
| `4:3`         | 1024×768  | 2048×1536 | ❌ Exceeds pixel limit       |
| `3:4`         | 768×1024  | 1536×2048 | ❌ Exceeds pixel limit       |
| `5:4`         | 1280×1024 | 2560×2048 | ❌ Exceeds pixel limit       |
| `4:5`         | 1024×1280 | 2048×2560 | ❌ Exceeds pixel limit       |
| `16:9`        | 1536×864  | 2048×1152 | **3840×2160** |
| `9:16`        | 864×1536  | 1152×2048 | **2160×3840** |
| `2:1`         | 2048×1024 | 2688×1344 | **3840×1920** |
| `1:2`         | 1024×2048 | 1344×2688 | **1920×3840** |
| `21:9`        | 2016×864  | 2688×1152 | **3840×1648** |
| `9:21`        | 864×2016  | 1152×2688 | **1648×3840** |

> `3:2` / `2:3` @ 2K are actually 2048×1360 (approximate ratio, error < 0.5%); 4K only supports 6 ratios (combinations exceeding OpenAI's total pixel limit are unavailable).

## Custom Size (size parameter)

When you need precise control over pixel dimensions, use the `size` parameter instead of the `aspect_ratio` + `resolution` combination.

**Valid range:** Total pixels 655,360 ~ 8,294,400 (upstream official limit)

**Tested effective lower limit:** Total pixels ≥ **2,088,960** (approximately 1440×1452 or 2048×1020 equivalent area)

Below this lower limit, the upstream does not report an error but silently replaces it with a close proportion size of about 1.57M pixels.

### Common size examples

| size value | Total pixels | Recommended use |
| --- | --- | --- |
| `2048x2048` | 4,194,304 | Square HD |
| `2048x1536` | 3,145,728 | 4:3 landscape |
| `1536x2048` | 3,145,728 | 3:4 portrait |
| `2048x1152` | 2,359,296 | 16:9 landscape |
| `1152x2048` | 2,359,296 | 9:16 portrait |
| `3840x2160` | 8,294,400 | 4K 16:9 (highest quality) |
| `2688x1344` | 3,612,672 | 2:1 wide landscape |

<Warning>
  **Do not pass together with `aspect_ratio` or `resolution`, otherwise a 400 will be returned.**
</Warning>
