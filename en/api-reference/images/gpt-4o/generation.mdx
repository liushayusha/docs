## Body

<ParamField body="model" type="string" default="gpt-4o-image" required>
  Name of the image generation model

  Example: `"gpt-4o-image"`
</ParamField>

<ParamField body="prompt" type="string" required>
  Text description for image generation

  Maximum 1000 characters
</ParamField>

<ParamField body="size" type="string">
  Size of the generated image

  Supported formats:

  * Aspect ratios: `1:1`, `2:3`, `3:2`
</ParamField>

<ParamField body="n" type="integer">
  Number of images to generate

  Supports 1, 2, 4; pre-charging applies based on the number generated

  Default value: 1

  **‚ö†Ô∏è Note:** Must input a pure number (e.g. `1`), do not add quotes, or an error will occur
</ParamField>

<ParamField body="image_urls" type="array">
  List of reference image URLs, used for image-to-image or image editing

  **üí° Quick fill (Try it area):**

  1. Click "+ Add an item" to add an image URL
  2. Input the full URL address of the image or base64 data

  <Expandable title="Detailed field description">
    <ParamField body="url" type="string" required>
      Image URL address

      Supports two formats:

      **1. Full image URL address**

      * Publicly accessible image URLs (http\:// or https\://)
      * Example: `https://example.com/image.jpg`

      **2. Base64 encoded format**

      * **Must use complete Data URI format**
      * Format: `data:image/{format};base64,{base64 data}`
      * Supported image formats: jpeg, png, webp, etc.
      * Example: `data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABg...`
      * ‚ö†Ô∏è Note: Must include the prefix `data:image/jpeg;base64,`

      **Limitations:**

      * Single image size no more than 10MB
      * Supported formats: .jpeg, .jpg, .png, .webp
    </ParamField>
  </Expandable>

  **Limit:** Up to 5 images
</ParamField>

<ParamField body="mask_url" type="string">
  URL of the mask image

  * Must be PNG format
  * Size must match the reference image
  * No larger than 4MB
</ParamField>
