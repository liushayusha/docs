## Body

<ParamField body="model" type="string" default="gpt-image-2-official" required>
  Image generation model name

  Fixed value: `gpt-image-2-official` (OpenAI official gpt-image-2 model)
</ParamField>

<ParamField body="prompt" type="string" required>
  Text description for image generation

  * Supports Chinese and English, detailed description is recommended
  * Before submission, it will go through platform sensitivity / safety review; violating content will return an error directly
</ParamField>

<ParamField body="size" type="string" default="1:1">
  Aspect ratio

  Public-facing ratio values; internally mapped to specific pixels by the system according to `resolution`.

  Supports 13 aspect ratios:

  * `1:1` - Square composition (default, social avatar / logo)
  * `3:2` - Landscape composition (common DSLR ratio)
  * `2:3` - Portrait composition (vertical poster)
  * `4:3` - Landscape composition (classic monitor / PPT)
  * `3:4` - Portrait composition
  * `5:4` - Landscape composition
  * `4:5` - Portrait composition (Instagram vertical post)
  * `16:9` - Landscape composition (widescreen video cover)
  * `9:16` - Portrait composition (full-screen mobile / short video cover)
  * `2:1` - Landscape composition (web banner)
  * `1:2` - Portrait composition
  * `21:9` - Landscape composition (cinematic ultrawide)
  * `9:21` - Portrait composition

  <Warning>
    Only ratio formats are supported; passing pixel dimensions (like `1024x1024`) will cause an immediate error
  </Warning>
</ParamField>

<ParamField body="resolution" type="string" default="1K">
  Resolution tier

  Controls the actual image clarity:

  * `1K` - Baseline 1024, economical and sufficient for daily use (default)
  * `2K` - Baseline 2048, suitable for posters / high-definition needs
  * `4K` - Baseline 3840, **only supported for 6 ratios** (`16:9` / `9:16` / `2:1` / `1:2` / `21:9` / `9:21`)

  <Warning>
    Unsupported 4K combinations will return 400:

    * `1:1` × `4K` ❌ (exceeds pixel limit)
    * `3:2` / `2:3` × `4K` ❌ (exceeds pixel limit)
    * `4:3` / `3:4` × `4K` ❌ (exceeds pixel limit)
    * `5:4` / `4:5` × `4K` ❌ (exceeds pixel limit)

    For HD versions of these ratios, use `resolution=2K` instead.
  </Warning>
</ParamField>

<ParamField body="quality" type="string" default="auto">
  Image quality

  * `auto` - Automatic (default, usually equivalent to `low`)
  * `low` - Fast and economical, sufficient outlines
  * `medium` - Balanced
  * `high` - Highest precision (4K + high takes >120 seconds)
</ParamField>

<ParamField body="image_urls" type="array">
  Reference image array; when provided, operates in image-to-image mode

  * Up to **9** reference images
  * Supports the following two formats:
    1. Publicly accessible URLs
    2. Base64 encoded format
       * Must use the full Data URI format: `data:image/{format};base64,{base64 data}`
       * Supported image formats: jpeg, png, webp
       * Example: `data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABg...`
       * ⚠️ Note: must include the `data:image/jpeg;base64,` prefix
</ParamField>

## Size × Resolution Mapping Table

`size × resolution` → Actual OpenAI pixels (13 ratios × 3 levels):

| size   | `1K`      | `2K`      | `4K`          |
| ------ | --------- | --------- | ------------- |
| `1:1`  | 1024×1024 | 2048×2048 | ❌ Pixel limit exceeded       |
| `3:2`  | 1536×1024 | 2048×1360 | ❌ Pixel limit exceeded       |
| `2:3`  | 1024×1536 | 1360×2048 | ❌ Pixel limit exceeded       |
| `4:3`  | 1024×768  | 2048×1536 | ❌ Pixel limit exceeded       |
| `3:4`  | 768×1024  | 1536×2048 | ❌ Pixel limit exceeded       |
| `5:4`  | 1280×1024 | 2560×2048 | ❌ Pixel limit exceeded       |
| `4:5`  | 1024×1280 | 2048×2560 | ❌ Pixel limit exceeded       |
| `16:9` | 1536×864  | 2048×1152 | **3840×2160** |
| `9:16` | 864×1536  | 1152×2048 | **2160×3840** |
| `2:1`  | 2048×1024 | 2688×1344 | **3840×1920** |
| `1:2`  | 1024×2048 | 1344×2688 | **1920×3840** |
| `21:9` | 2016×864  | 2688×1152 | **3840×1648** |
| `9:21` | 864×2016  | 1152×2688 | **1648×3840** |

> `3:2` / `2:3` at 2K is actually 2048×1360 (approximate ratio, error < 0.5%); 4K only supports 6 ratios (combinations exceeding the OpenAI total pixel limit are unavailable).

## Query Task Result

After a successful submission, a `task_id` is returned. You can poll the task status via `GET /v1/tasks/{task_id}`. See [Task Query API](/en/api-reference/tasks/get-task) for details.

### Polling Recommendations

* **Initial Query Delay**: Wait 10~20 seconds after submission before starting to query
* **Query Interval**: Recommended every 3~5 seconds
* **Timeout Reference**: The `high + 2K/4K` combination may take up to 130 seconds, client timeout is recommended to be ≥ 180 seconds
