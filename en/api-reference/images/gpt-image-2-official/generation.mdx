## Body

<ParamField body="model" type="string" default="gpt-image-2-official" required>
  Image generation model name

  Fixed value `gpt-image-2-official` (OpenAI official gpt-image-2 model)
</ParamField>

<ParamField body="params" type="object" required>
  Model parameters object

  <Expandable title="params Object Properties">
    <ParamField body="prompt" type="string" required>
      Text description for image generation

      * Supports Chinese and English, detailed description recommended
      * Content will be checked against platform sensitive words / security review before submission; hits on violations will return an error directly
    </ParamField>

    <ParamField body="size" type="string" default="1:1">
      Aspect ratio

      Externally uses ratio values; internally mapped automatically to specific pixels by `resolution`.

      Supports 13 ratios:

      * `1:1` - Square composition (default, social avatar / logo)
      * `3:2` - Landscape (common DSLR ratio)
      * `2:3` - Portrait (poster vertical)
      * `4:3` - Landscape (classic monitor / PPT)
      * `3:4` - Portrait
      * `5:4` - Landscape
      * `4:5` - Portrait (Instagram vertical post)
      * `16:9` - Landscape (widescreen video cover)
      * `9:16` - Portrait (phone fullscreen / short video cover)
      * `2:1` - Landscape (web banner)
      * `1:2` - Portrait
      * `21:9` - Landscape (cinema ultra widescreen)
      * `9:21` - Portrait

      <Warning>
        Only ratio format is supported; passing pixel dimensions (e.g. `1024x1024`) will cause an error
      </Warning>
    </ParamField>

    <ParamField body="resolution" type="string" default="1K">
      Resolution tier

      Controls actual output image clarity:

      * `1K` - 1024 baseline, cost-saving daily use (default)
      * `2K` - 2048 baseline, suitable for posters / HD needs
      * `4K` - 3840 baseline, **only supported for 6 ratios** (`16:9` / `9:16` / `2:1` / `1:2` / `21:9` / `9:21`)

      <Warning>
        Unsupported 4K combinations will return 400:

        * `1:1` × `4K` ❌ (exceeds pixel limit)
        * `3:2` / `2:3` × `4K` ❌ (exceeds pixel limit)
        * `4:3` / `3:4` × `4K` ❌ (exceeds pixel limit)
        * `5:4` / `4:5` × `4K` ❌ (exceeds pixel limit)

        For HD images of these ratios, use `resolution=2K` instead.
      </Warning>
    </ParamField>

    <ParamField body="quality" type="string" default="auto">
      Image quality

      * `auto` - Automatic (default)
      * `low` - Fast and cost-saving, outlines are sufficient
      * `medium` - Balanced
      * `high` - Highest precision (4K + high takes >120 seconds)
    </ParamField>

    <ParamField body="image_urls" type="array">
      Reference image array; when provided, image-to-image generation mode is used

      * Up to **9** reference images
      * Supports the following two formats:
        1. Public accessible URLs
        2. Base64 encoded format
           * Must use the full Data URI format: `data:image/{format};base64,{base64data}`
           * Supported image formats: jpeg, png, webp
           * Example: `data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABg...`
           * ⚠️ Note: must include the `data:image/jpeg;base64,` prefix
    </ParamField>
  </Expandable>
</ParamField>

## Size × Resolution Mapping Table

`size × resolution` → Actual OpenAI pixels (13 ratios × 3 levels):

| size   | `1K`      | `2K`      | `4K`          |
| ------ | --------- | --------- | ------------- |
| `1:1`  | 1024×1024 | 2048×2048 | ❌ Pixel limit exceeded       |
| `3:2`  | 1536×1024 | 2048×1360 | ❌ Pixel limit exceeded       |
| `2:3`  | 1024×1536 | 1360×2048 | ❌ Pixel limit exceeded       |
| `4:3`  | 1024×768  | 2048×1536 | ❌ Pixel limit exceeded       |
| `3:4`  | 768×1024  | 1536×2048 | ❌ Pixel limit exceeded       |
| `5:4`  | 1280×1024 | 2560×2048 | ❌ Pixel limit exceeded       |
| `4:5`  | 1024×1280 | 2048×2560 | ❌ Pixel limit exceeded       |
| `16:9` | 1536×864  | 2048×1152 | **3840×2160** |
| `9:16` | 864×1536  | 1152×2048 | **2160×3840** |
| `2:1`  | 2048×1024 | 2688×1344 | **3840×1920** |
| `1:2`  | 1024×2048 | 1344×2688 | **1920×3840** |
| `21:9` | 2016×864  | 2688×1152 | **3840×1648** |
| `9:21` | 864×2016  | 1152×2688 | **1648×3840** |

> `3:2` / `2:3` at 2K is actually 2048×1360 (approximate ratio, error < 0.5%); 4K only supports 6 ratios (combinations exceeding the OpenAI total pixel limit are unavailable).

## Query Task Result

After successful submission, a `task_id` is returned. Poll the task status through `GET /v1/tasks/{task_id}`. For details, see [Task Query Endpoint](/en/api-reference/tasks/get-task).

### Polling Recommendations

* **Initial query delay**: Wait 10~20 seconds after submission before starting to query
* **Query interval**: Recommended every 3~5 seconds
* **Timeout reference**: The `high + 2K/4K` combination can take up to 130 seconds; client timeout is recommended to be ≥ 180 seconds
