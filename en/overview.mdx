## API Workflow

### 1. Submit Generation Task

All image and video generation are processed asynchronously. When submitting a task, you need to provide the `out_task_id` (the initiator's task ID). The system will return both `task_id` and `out_task_id`:

```bash
curl --request POST \
  --url https://aireiter.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "A cute cat in the garden",
      "aspect_ratio": "1:1"
    },
    "out_task_id": "my_task_123456"
  }'
```

### 2. Query Task Status

Use the returned `task_id` or the submitted `out_task_id` to query the task progress and results (either one):

```bash
# Query using task_id
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'

# Or query using out_task_id
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "out_task_id": "my_task_123456"
  }'
```

### 3. Retrieve Generation Results

After the task is completed, the response will include URLs of the generated images or videos:

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "out_task_id": "my_task_123456",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## Core Features

### Asynchronous Task Processing

All generation requests use an asynchronous processing mode to avoid long wait times:

- When submitting a task, you must provide `out_task_id` (the initiator’s task ID for business identification)
- The system immediately returns the generated `task_id` and your submitted `out_task_id`
- You can query the task status using either `task_id` or `out_task_id` (choose one)
- Supports optional webhook callback notifications
- Task status updates in real-time: `pending` → `processing` → `completed`

### Multi-Model Support

A unified API endpoint supports multiple image and video generation models:

- **Image Generation**: 5 models including GPT-4o, Gemini, Seedream
- **Video Generation**: 4 models including Sora2, VEO3
- Unified request format and response structure
- Flexible parameter configuration

### Transparent Pricing

Charged by points, clear and transparent:

- Each task returns an estimated points consumption
- Actual points consumption is shown after task completion
- You only pay for successfully generated content
- No hidden fees

### Flexible Configuration

Supports various generation parameter configurations:

- **Image**: multiple aspect ratios (1:1, 3:2, 2:3)
- **Video**: multiple aspect ratios (16:9, 9:16), durations (10 seconds, 15 seconds), resolutions
- Supports reference images (image-to-image, image-to-video)

## FAQ

<AccordionGroup>
  <Accordion title="How to get an API Key?">
    Visit the <a href={apiKeyUrl} target="_blank">API Key Management Page</a>, where you can create and manage your API Key after logging in.
  </Accordion>

  <Accordion title="How long does it take to generate images and videos?">
    All generation tasks are processed asynchronously. Image generation usually takes a few seconds to tens of seconds, while video generation depends on duration and resolution, typically taking 1-5 minutes. After submitting a task, you will immediately receive a task_id, which you can use to track progress via the status query API.
  </Accordion>

  <Accordion title="How to query task status?">
    Use the task status query API by providing the `task_id` returned when submitting the task, or your own `out_task_id` (either one):

    ```bash
    POST https://aireiter.com/api/openapi/query
    ```

    See the [Task Status Query Documentation](/en/api-reference/tasks/status) for details.
  </Accordion>

  <Accordion title="What is out_task_id?">
    `out_task_id` is the client-side task ID, which you customize when submitting a task:

    - **Required parameter**: `out_task_id` must be provided when submitting a task
    - **Business identifier**: You can use an order number, business ID, etc., from your system as the identifier
    - **Flexible query**: You can directly query task status using `out_task_id` without saving the system-returned `task_id`
    - **Result association**: Both `task_id` and `out_task_id` will be returned in query results to help associate with your business data
  </Accordion>

  <Accordion title="How is billing done?">
    AIReiter uses a credit-based billing system:

    - An estimated credit consumption (`estimated_credits`) is returned when submitting a task
    - Actual credits used (`credits_used`) are returned after the task is completed
    - You only pay for successfully generated content; failed tasks are not charged
    - Credits consumed vary by model
  </Accordion>

  <Accordion title="What image and video formats are supported?">
    - **Images**: Supports common formats such as JPEG, PNG
    - **Videos**: Outputs in MP4 format
    - **Reference images**: Input supports publicly accessible image URLs
  </Accordion>
</AccordionGroup>
