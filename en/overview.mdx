## API Workflow

### 1. Submit a Generate Task

All image and video generation use an asynchronous processing mode. When submitting a task, you need to pass `out_task_id` (initiator task ID). The system will return `task_id` and `out_task_id`:

```bash
curl --request POST \
  --url https://test.aireiter.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "A cute cat in the garden",
      "aspect_ratio": "1:1"
    },
    "out_task_id": "my_task_123456"
  }'
```

### 2. Query Task Status

Use the returned `task_id` or the `out_task_id` you submitted to check the task progress and result (either one):

```bash
# Query by task_id
curl --request POST \
  --url https://test.aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'

# Or query by out_task_id
curl --request POST \
  --url https://test.aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "out_task_id": "my_task_123456"
  }'
```

### 3. Get Generate Result

After the task is completed, the response will include the generated image or video URL:

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "out_task_id": "my_task_123456",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## Core Features

### Asynchronous Task Processing

All generation requests use an asynchronous processing model to avoid long waiting times:

- When submitting a task, provide `out_task_id` (initiator's task ID, used for business identification)
- Immediately return the system-generated `task_id` and your submitted `out_task_id`
- You can query the task status using either `task_id` or `out_task_id`
- Supports webhook callback notifications (optional)
- Task status updates in real time: `pending` → `processing` → `completed`

### Multi-Model Support

A unified API endpoint supports various image and video generation models:

- **Image Generation**: 5 models including GPT-4o, Gemini, Seedream
- **Video Generation**: 4 models including Sora2, VEO3
- Unified request format and response structure
- Flexible parameter configuration

### Transparent Pricing

Charged by credits, clear and transparent:

- Each task returns an estimated credit consumption
- Actual credit consumption shown after task completion
- Pay only for successfully generated content
- No hidden fees

### Flexible Configuration

Supports various generation parameter settings:

- **Images**: multiple aspect ratios (1:1, 3:2, 2:3)
- **Videos**: multiple aspect ratios (16:9, 9:16), durations (10 seconds, 15 seconds), resolutions
- Supports reference images (image-to-image, image-to-video)
- Custom metadata

## FAQ

<AccordionGroup>
  <Accordion title="How to get the API Key?">
    Visit the <a href={apiKeyUrl} target="_blank">API Key management page</a>, log in to create and manage your API Keys.
  </Accordion>

  <Accordion title="How long does it take to generate images and videos?">
    All generation tasks are processed asynchronously. Image generation usually takes a few seconds to tens of seconds. Video generation typically takes 1-5 minutes depending on duration and resolution. After submitting a task, a task_id is returned immediately, and you can track progress through the status query API.
  </Accordion>

  <Accordion title="How to query the task status?">
    Use the task status query API by passing the `task_id` returned when submitting the task, or your own set `out_task_id` (either one):

    ```bash
    POST https://test.aireiter.com/api/openapi/query
    ```

    See the [Task Status Query Documentation](/en/api-reference/tasks/status) for details.
  </Accordion>

  <Accordion title="What is out_task_id?">
    `out_task_id` is the initiator’s task ID, which you customize when submitting the task:

    - **Required parameter**: must include `out_task_id` when submitting the task
    - **Business identifier**: can use your system’s order number, business ID, etc. as the identifier
    - **Flexible querying**: you can use `out_task_id` directly to query task status without saving the system returned `task_id`
    - **Result association**: both `task_id` and `out_task_id` are returned in the query result to help link your business data
  </Accordion>

  <Accordion title="Is webhook callback supported?">
    Yes, you can pass the `callback_url` parameter when submitting the task. After completion, the system will send a POST request to that URL containing the full task result.
  </Accordion>

  <Accordion title="How is billing handled?">
    AIReiter uses a credit-based billing system:

    - The estimated credit consumption (`estimated_credits`) is returned when submitting a task
    - The actual credits used (`credits_used`) are returned after task completion
    - You only pay for successfully generated content; failed tasks are free
    - Credit consumption varies by model
  </Accordion>

  <Accordion title="Which image and video formats are supported?">
    - **Images**: supports common formats such as JPEG and PNG
    - **Videos**: outputs in MP4 format
    - **Reference images**: input supports publicly accessible image URLs
  </Accordion>
</AccordionGroup>
