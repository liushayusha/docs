## API Workflow

### 1. Submit Generation Task

All image and video generation uses asynchronous processing. After submitting a task, the system immediately returns a `task_id`:

```bash
curl --request POST \
  --url https://pxz-api.xinsulv.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "一只可爱的猫咪在花园里",
      "aspect_ratio": "1:1"
    }
  }'
```

### 2. Query Task Status

Use the returned `task_id` to query task progress and results:

```bash
curl --request POST \
  --url https://pxz-api.xinsulv.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'
```

### 3. Get Generation Results

Once the task is completed, the response will contain the generated image or video URL:

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## Core Features

### Asynchronous Task Processing

All generation requests use asynchronous processing to avoid long waiting times:

- Immediately returns `task_id`
- Query progress through task status API
- Support webhook callback notifications (optional)
- Real-time task status updates: `pending` → `processing` → `completed`

### Multi-Model Support

A unified API endpoint supporting multiple image and video generation models:

- **Image Generation**: 5 models including GPT-4o, Gemini, Seedream
- **Video Generation**: 4 models including Sora2, VEO3
- Unified request format and response structure
- Flexible parameter configuration

### Transparent Pricing

Credit-based billing, transparent and clear:

- Each task returns estimated credit consumption
- Actual credits used displayed after task completion
- Only pay for successfully generated content
- No hidden fees

### Flexible Configuration

Supports various generation parameter configurations:

- **Images**: Multiple aspect ratios (1:1, 3:2, 2:3)
- **Videos**: Multiple aspect ratios (16:9, 9:16), durations (10s, 15s), resolutions
- Support reference images (image-to-image, image-to-video)
- Custom metadata

## Frequently Asked Questions

<AccordionGroup>
  <Accordion title="How do I get an API Key?">
    Visit the [API Key Management Page](https://apimart.ai/console/token), and you can create and manage your API Key after logging in.
  </Accordion>

  <Accordion title="How long does image and video generation take?">
    All generation tasks are processed asynchronously. Image generation typically takes a few seconds to tens of seconds, while video generation usually takes 1-5 minutes depending on duration and resolution. After submitting a task, you'll immediately receive a task_id, and you can track progress through the status query API.
  </Accordion>

  <Accordion title="How do I query task status?">
    Use the task status query API with the `task_id` returned when submitting the task:

    ```bash
    POST https://pxz-api.xinsulv.com/api/openapi/query
    ```

    See the [Task Status Query Documentation](/en/api-reference/tasks/status) for details.
  </Accordion>

  <Accordion title="Do you support webhook callbacks?">
    Yes, you can pass the `callback_url` parameter when submitting a task. After the task is completed, the system will send a POST request to that address containing the complete task results.
  </Accordion>

  <Accordion title="How does billing work?">
    APIMart uses a credit-based billing system:

    - Estimated credit consumption (`estimated_credits`) is returned when submitting a task
    - Actual credits used (`credits_used`) is returned after task completion
    - Only pay for successfully generated content, failed tasks are not charged
    - Different models have different credit consumption rates
  </Accordion>

  <Accordion title="What image and video formats are supported?">
    - **Images**: Supports common formats like JPEG, PNG
    - **Videos**: Output in MP4 format
    - **Reference Images**: Input supports publicly accessible image URLs
  </Accordion>
</AccordionGroup>
