## API ワークフロー

### 1. 生成タスクの送信

すべての画像および動画生成は非同期処理モードを採用しています。タスクを送信する際には `out_task_id`（発行元タスクID）を渡す必要があり、システムは `task_id` と `out_task_id` を返します：

```bash
curl --request POST \
  --url https://aireiter.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "かわいい猫が庭にいます",
      "aspect_ratio": "1:1"
    },
    "out_task_id": "my_task_123456"
  }'
```

### 2. タスクステータスの照会

返された `task_id` または送信時の `out_task_id` を使ってタスクの進行状況や結果を照会します（どちらか一方を使用）：

```bash
# task_id で照会
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'

# または out_task_id で照会
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "out_task_id": "my_task_123456"
  }'
```

### 3. 生成結果の取得

タスク完了後、レスポンスには生成された画像または動画のURLが含まれます：

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "out_task_id": "my_task_123456",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## コア機能

### 非同期タスク処理

すべての生成リクエストは非同期処理モードを採用し、長時間の待機を回避します：

- タスクを送信する際、`out_task_id`（発行者側のタスクID、ビジネス識別用）を渡す必要があります
- システムが生成した`task_id`と送信した`out_task_id`を即座に返します
- タスクの状態は`task_id`または`out_task_id`で照会可能（どちらか一方）
- webhookコールバック通知をサポート（任意）
- タスクの状態はリアルタイムで更新：`pending` → `processing` → `completed`

### 複数モデル対応

統一されたAPIエンドポイントで、複数の画像・動画生成モデルをサポート：

- **画像生成**：GPT-4o、Gemini、Seedreamなど5モデル
- **動画生成**：Sora2、VEO3など4モデル
- 統一されたリクエスト形式とレスポンス構造
- 柔軟なパラメータ設定

### 透明な料金設定

ポイント制で課金され、透明かつ明確です：

- 各タスクに対して予想消費ポイントを返却
- タスク完了後に実際の消費ポイントを表示
- 成功した生成コンテンツのみ課金
- 隠れた料金なし

### 柔軟な設定

複数の生成パラメータ設定をサポート：

- **画像**：複数のアスペクト比（1:1、3:2、2:3）
- **動画**：複数のアスペクト比（16:9、9:16）、長さ（10秒、15秒）、解像度
- 参照画像のサポート（画像から画像、画像から動画）
- カスタムメタデータ（metadata）

<AccordionGroup>
  <Accordion title="APIキーはどのように取得しますか？">
    <a href={apiKeyUrl} target="_blank">APIキー管理ページ</a>にアクセスし、ログイン後にAPIキーの作成と管理ができます。
  </Accordion>

  <Accordion title="画像および動画の生成にはどれくらい時間がかかりますか？">
    すべての生成タスクは非同期で処理されます。画像生成は通常数秒から数十秒、動画生成は長さや解像度により、通常1〜5分かかります。タスクを提出するとすぐに task_id が返され、ステータス照会APIで進行状況を追跡できます。
  </Accordion>

  <Accordion title="タスクのステータスはどのように確認しますか？">
    タスクステータス照会APIを使い、提出時に返された `task_id` か、ご自分で設定した `out_task_id`（いずれか一方）を渡します：

    ```bash
    POST https://aireiter.com/api/openapi/query
    ```

    詳細は [タスクステータス照会ドキュメント](/ja/api-reference/tasks/status) をご覧ください。
  </Accordion>

  <Accordion title="out_task_idとは何ですか？">
    `out_task_id` は依頼側のタスクIDで、タスク提出時にカスタム設定します：

    - **必須パラメータ**：タスク提出時に必ず `out_task_id` を渡してください
    - **業務識別**：システム内の注文番号や業務IDなどを識別子として使えます
    - **柔軟な照会**：システムから返される `task_id` を保存しなくても、`out_task_id` で直接タスクステータスを調べられます
    - **結果の連携**：照会結果には `task_id` と `out_task_id` の両方が返るため、業務データとの紐付けが容易です
  </Accordion>

  <Accordion title="Webhookコールバックは対応していますか？">
    はい。タスク提出時に `callback_url` パラメータを渡すことが可能です。タスク完了後、システムはそのURLにPOSTリクエストを送り、タスクの結果をすべて含みます。
  </Accordion>

  <Accordion title="料金はどのように計算されますか？">
    AIReiterはポイント制料金体系を採用しています：

    - タスク提出時に推定消費ポイント（`estimated_credits`）が返されます
    - タスク完了後に実際の消費ポイント（`credits_used`）が返されます
    - 成功した生成内容にのみ課金され、失敗したタスクは無料です
    - モデルによってポイント消費が異なります
  </Accordion>

  <Accordion title="対応している画像フォーマットと動画フォーマットは？">
    - **画像**：JPEG、PNGなどの一般的なフォーマットに対応
    - **動画**：出力はMP4フォーマット
    - **参照画像**：入力には公開アクセス可能な画像URLが使用可能
  </Accordion>
</AccordionGroup>
