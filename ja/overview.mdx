## API ワークフロー

### 1. 生成タスクの送信

すべての画像および動画生成は非同期処理モードを採用しています。タスクを送信する際には `out_task_id`（発行元タスクID）を渡す必要があり、システムは `task_id` と `out_task_id` を返します：

```bash
curl --request POST \
  --url https://aireiter.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "かわいい猫が庭にいます",
      "aspect_ratio": "1:1"
    },
    "out_task_id": "my_task_123456"
  }'
```

### 2. タスクステータスの照会

返された `task_id` または送信時の `out_task_id` を使ってタスクの進行状況や結果を照会します（どちらか一方を使用）：

```bash
# task_id で照会
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'

# または out_task_id で照会
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "out_task_id": "my_task_123456"
  }'
```

### 3. 生成結果の取得

タスク完了後、レスポンスには生成された画像または動画のURLが含まれます：

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "out_task_id": "my_task_123456",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## コア機能

### 非同期タスク処理

すべての生成リクエストは非同期処理モードを採用し、長時間の待機を回避します：

- タスクを送信する際に `out_task_id`（発信元のタスクID、ビジネス識別用）を渡す必要があります
- システム生成の `task_id` と送信した `out_task_id` を即時に返します
- `task_id` または `out_task_id` でタスクの状態を照会可能（二者選択）
- webhook コールバック通知に対応（任意）
- タスク状態はリアルタイムに更新：`pending` → `processing` → `completed`

### マルチモデル対応

統一されたAPIエンドポイントで、複数の画像および動画生成モデルをサポート：

- **画像生成**：GPT-4o、Gemini、Seedreamなど5つのモデル
- **動画生成**：Sora2、VEO3など4つのモデル
- 統一されたリクエストフォーマットとレスポンス構造
- 柔軟なパラメータ設定

### 透明な料金

ポイント課金で透明かつ明瞭：

- 各タスクの予測消費ポイントを返します
- タスク完了後に実際の消費ポイントを表示
- 成功して生成されたコンテンツだけに料金が発生
- 隠れた費用はありません

### 柔軟な設定

多様な生成パラメータ設定をサポート：

- **画像**：複数のアスペクト比（1:1、3:2、2:3）
- **動画**：複数のアスペクト比（16:9、9:16）、時間（10秒、15秒）、解像度
- 参考画像のサポート（画像→画像、画像→動画）

  <Accordion title="APIキーはどのように取得しますか？">
    <a href={apiKeyUrl} target="_blank">APIキー管理ページ</a>にアクセスし、ログイン後にAPIキーを作成および管理できます。

  <Accordion title="画像と動画の生成にはどのくらい時間がかかりますか？">
    すべての生成タスクは非同期で処理されます。画像生成は通常数秒から数十秒かかり、動画生成は長さと解像度によって異なり、通常1〜5分かかります。タスクを送信するとすぐにtask_idが返され、状態照会APIで進捗を追跡できます。
  </Accordion>

  <Accordion title="タスクの状態はどのように確認しますか？">
    タスク状態照会APIを使用し、送信時に返された`task_id`またはご自身で設定した`out_task_id`（いずれか一方）を渡します：

    ```bash
    POST https://aireiter.com/api/openapi/query
    ```

    詳細は[タスク状態照会ドキュメント](/en/api-reference/tasks/status)をご覧ください。
  </Accordion>

  <Accordion title="out_task_idとは何ですか？">
    `out_task_id`は発信側タスクIDで、タスク送信時にカスタム設定します：

    - **必須パラメータ**：タスク送信時に`out_task_id`を必ず渡してください
    - **業務識別**：システム内の注文番号や業務IDなどを識別子として利用可能
    - **柔軟な照会**：`out_task_id`を使って直接タスク状態を照会でき、システムが返す`task_id`を保存する必要はありません
    - **結果連携**：照会結果には`task_id`と`out_task_id`の両方が返され、業務データと連携しやすいです
  </Accordion>

  <Accordion title="課金方法は？">
    AIReiterはポイント制課金を採用しています：

    - タスク送信時に推定消費ポイント（`estimated_credits`）を返します
    - タスク完了後に実際の消費ポイント（`credits_used`）を返します
    - 成功して生成された内容に対してのみ課金され、失敗したタスクは無料です
    - モデルによってポイント消費が異なります
  </Accordion>

  <Accordion title="対応している画像・動画フォーマットは何ですか？">
    - **画像**：JPEG、PNGなどの一般的なフォーマットをサポート
    - **動画**：MP4形式で出力
    - **参照画像**：入力には公開アクセス可能な画像URLをサポート
  </Accordion>
</AccordionGroup>
