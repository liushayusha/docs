## API ワークフロー

### 1. 生成タスクの送信

すべての画像および動画の生成は非同期処理モードを採用しています。タスクを送信する際に `out_task_id`（発注者タスクID）を渡す必要があり、システムは `task_id` と `out_task_id` を返します：

```bash
curl --request POST \
  --url https://test.aireiter.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "かわいい猫が庭にいる",
      "aspect_ratio": "1:1"
    },
    "out_task_id": "my_task_123456"
  }'
```

### 2. タスク状態の照会

返された `task_id` または送信時の `out_task_id` を使って、タスクの進行状況と結果を照会します（どちらか一方で可）：

```bash
# task_idを使用して照会
curl --request POST \
  --url https://test.aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'

# またはout_task_idを使用して照会
curl --request POST \
  --url https://test.aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "out_task_id": "my_task_123456"
  }'
```

### 3. 生成結果の取得

タスク完了後、レスポンスに生成された画像または動画のURLが含まれます：

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "out_task_id": "my_task_123456",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## コア機能

### 非同期タスク処理

すべての生成リクエストは非同期処理モードを採用し、長時間の待機を避けます：

- タスクを送信する際に `out_task_id`（発信側タスクID、ビジネス識別用）を渡す必要があります
- システムが生成した `task_id` と送信した `out_task_id` を即座に返します
- `task_id` または `out_task_id` でタスクの状態を照会可能（二者択一）
- webhookによるコールバック通知をサポート（任意）
- タスク状態はリアルタイムで更新されます：`pending` → `processing` → `completed`

### マルチモデル対応

統一されたAPIエンドポイントで、複数の画像・動画生成モデルに対応：

- **画像生成**：GPT-4o、Gemini、Seedream など5つのモデル
- **動画生成**：Sora2、VEO3 など4つのモデル
- 統一されたリクエストフォーマットとレスポンス構造
- 柔軟なパラメータ設定

### 透明な価格設定

ポイント課金制で明確かつ透明：

- 各タスクは消費予測ポイントを返します
- タスク完了後に実消費ポイントを表示
- 成功して生成されたコンテンツのみ課金対象
- 隠れた料金なし

### 柔軟な設定

多様な生成パラメータの設定をサポート：

- **画像**：多様なアスペクト比（1:1、3:2、2:3）
- **動画**：多様なアスペクト比（16:9、9:16）、長さ（10秒、15秒）、解像度
- 参照画像をサポート（画像から画像、画像から動画）
- カスタムメタデータ(metadata)

## 認証方式

<AccordionGroup>
  <Accordion title="APIキーはどのように取得しますか？">
    <a href={apiKeyUrl} target="_blank">APIキー管理ページ</a>にアクセスし、ログイン後にAPIキーの作成および管理が可能です。
  </Accordion>

  <Accordion title="画像や動画の生成にはどのくらい時間がかかりますか？">
    すべての生成タスクは非同期で処理されます。画像生成は通常数秒から数十秒かかり、動画生成は長さや解像度によって異なり、通常1～5分かかります。タスクを提出するとすぐにtask_idが返され、状態照会APIを使って進行状況を追跡できます。
  </Accordion>

  <Accordion title="タスクの状態はどのように照会しますか？">
    タスク状態照会APIを使用すると、提出時に返された`task_id`または自身で設定した`out_task_id`（どちらか一方）を渡して照会できます：

    ```bash
    POST https://test.aireiter.com/api/openapi/query
    ```

    詳細は[タスク状態照会ドキュメント](/jp/api-reference/tasks/status)をご覧ください。
  </Accordion>

  <Accordion title="out_task_idとは何ですか？">
    `out_task_id`は発行者側のタスクIDで、タスク提出時にカスタム設定します：

    - **必須パラメータ**：タスク提出時に`out_task_id`を必ず渡す必要があります
    - **業務識別**：自社システムの注文番号や業務IDなどで識別できます
    - **照会の柔軟性**：`out_task_id`で直接タスク状態を照会でき、システム返却の`task_id`保存は不要です
    - **結果連携**：照会結果に`task_id`と`out_task_id`の両方が返されるため、業務データの連携が容易です
  </Accordion>

  <Accordion title="Webhookコールバックに対応していますか？">
    はい、タスク提出時に`callback_url`パラメータを指定できます。タスク完了後、システムがそのURLへPOSTリクエストを送り、完全なタスク結果を含みます。
  </Accordion>

  <Accordion title="課金はどのように行われますか？">
    AIReiterはポイント制課金を採用しています：

    - タスク提出時に推定消費ポイント（`estimated_credits`）を返します
    - タスク完了後に実際の消費ポイント（`credits_used`）を返します
    - 成功した生成内容に対してのみ課金し、失敗タスクは無料です
    - モデルによってポイント消費は異なります
  </Accordion>

  <Accordion title="対応している画像フォーマットと動画フォーマットは何ですか？">
    - **画像**：JPEG、PNGなどの一般的なフォーマットに対応
    - **動画**：出力はMP4フォーマット
    - **参照画像**：公開アクセス可能な画像URLを入力としてサポート
  </Accordion>
</AccordionGroup>
