## Body

<ParamField body="model" type="string" default="gpt-image-2-official" required>
  画像生成モデルの名前

  固定で `gpt-image-2-official` を入力してください（OpenAI公式のgpt-image-2モデル）
</ParamField>

<ParamField body="params" type="object" required>
  モデルのパラメータオブジェクト

  <Expandable title="params オブジェクトのプロパティ">
    <ParamField body="prompt" type="string" required>
      画像生成のテキスト説明

      * 日本語と英語をサポートし、詳細な説明を推奨します
      * 提出前にプラットフォームの禁止ワード／セキュリティチェックが行われ、違反内容が検出されると直接エラーが返されます
    </ParamField>

    <ParamField body="size" type="string" default="1:1">
      画面比率

      外部には比率の値を使用し、システム内部では `resolution` により具体的なピクセルに自動マッピングされます。

      13種類の比率をサポートします：

      * `1:1` - 正方形構図（デフォルト、SNSのプロフィール写真／ロゴ用）
      * `3:2` - 横構図（一眼レフカメラで一般的な比率）
      * `2:3` - 縦構図（ポスター縦長）
      * `4:3` - 横構図（クラシックなモニター／PPT用）
      * `3:4` - 縦構図
      * `5:4` - 横構図
      * `4:5` - 縦構図（Instagram縦型投稿）
      * `16:9` - 横構図（ワイドスクリーン動画のサムネイル）
      * `9:16` - 縦構図（スマホ全画面／ショート動画サムネイル）
      * `2:1` - 横構図（ウェブのバナー用）
      * `1:2` - 縦構図
      * `21:9` - 横構図（映画の超ワイドスクリーン）
      * `9:21` - 縦構図

      <Warning>
        比率の表記のみサポートし、ピクセルサイズ（例：`1024x1024`）を送信するとエラーになります
      </Warning>
    </ParamField>

    <ParamField body="resolution" type="string" default="1K">
      解像度レベル

      実際の出力画像の鮮明度を制御します：

      * `1K` - ベース1024、日常利用で経済的（デフォルト）
      * `2K` - ベース2048、ポスター／高精細ニーズに適合
      * `4K` - ベース3840、**6つの比率のみ対応**（`16:9` / `9:16` / `2:1` / `1:2` / `21:9` / `9:21`）

      <Warning>
        サポート外の4K組み合わせは400エラーを返します：

        * `1:1` × `4K` ❌（ピクセル上限超過）
        * `3:2` / `2:3` × `4K` ❌（ピクセル上限超過）
        * `4:3` / `3:4` × `4K` ❌（ピクセル上限超過）
        * `5:4` / `4:5` × `4K` ❌（ピクセル上限超過）

        上記比率で高精細を希望する場合は `resolution=2K` を指定してください。
      </Warning>
    </ParamField>

    <ParamField body="quality" type="string" default="auto">
      画像品質

      * `auto` - 自動（デフォルト）
      * `low` - 高速・節約、アウトラインに十分
      * `medium` - バランス
      * `high` - 最高精度（4K + high は処理時間 >120秒）
    </ParamField>

    <ParamField body="image_urls" type="array">
      参考画像の配列、渡すと画像ベース生成モードになります

      * 最大 **9枚** の参考画像
      * 以下の2形式をサポート：
        1. 公開アクセス可能なURL
        2. Base64エンコード形式
           * 完全なData URI形式を使用する必要があります：`data:image/{フォーマット};base64,{base64データ}`
           * サポート画像フォーマット：jpeg、png、webp
           * 例：`data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABg...`
           * ⚠️ 注意：`data:image/jpeg;base64,`のプレフィックス部分を必ず含めてください
    </ParamField>
  </Expandable>
</ParamField>

## サイズ × 解像度マッピング表

`size × resolution` → OpenAI 実際のピクセル（13 比率 × 3 画質レベル）：

| size   | `1K`      | `2K`      | `4K`          |
| ------ | --------- | --------- | ------------- |
| `1:1`  | 1024×1024 | 2048×2048 | ❌ 超ピクセル上限       |
| `3:2`  | 1536×1024 | 2048×1360 | ❌ 超ピクセル上限       |
| `2:3`  | 1024×1536 | 1360×2048 | ❌ 超ピクセル上限       |
| `4:3`  | 1024×768  | 2048×1536 | ❌ 超ピクセル上限       |
| `3:4`  | 768×1024  | 1536×2048 | ❌ 超ピクセル上限       |
| `5:4`  | 1280×1024 | 2560×2048 | ❌ 超ピクセル上限       |
| `4:5`  | 1024×1280 | 2048×2560 | ❌ 超ピクセル上限       |
| `16:9` | 1536×864  | 2048×1152 | **3840×2160** |
| `9:16` | 864×1536  | 1152×2048 | **2160×3840** |
| `2:1`  | 2048×1024 | 2688×1344 | **3840×1920** |
| `1:2`  | 1024×2048 | 1344×2688 | **1920×3840** |
| `21:9` | 2016×864  | 2688×1152 | **3840×1648** |
| `9:21` | 864×2016  | 1152×2688 | **1648×3840** |

> `3:2` / `2:3` @ 2K は実際2048×1360（近似比率、誤差 \< 0.5%）；4K は6つの比率のみ対応（総ピクセルがOpenAI上限を超える組み合わせは使用不可）。

## タスク結果の取得

送信成功後に `task_id` が返されます。 `GET /v1/tasks/{task_id}` でタスクの状態をポーリングしてください。詳細は [タスク取得エンドポイント](/jp/api-reference/tasks/get-task) を参照してください。

### ポーリングの推奨

* **初回取得遅延**：送信後 10〜20 秒待ってから取得を開始してください
* **取得間隔**：3〜5 秒に1回を推奨します
* **タイムアウト目安**：`high + 2K/4K` の組み合わせでは最大 130 秒かかるため、クライアントのタイムアウトは 180 秒以上を推奨します
