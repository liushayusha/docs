## Body

<ParamField body="model" type="string" default="gpt-image-2-official" required>
  画像生成モデル名

  固定で `gpt-image-2-official` を入力（OpenAI公式の gpt-image-2 モデル）
</ParamField>

<ParamField body="prompt" type="string" required>
  画像生成用のテキスト説明

  * 日本語・英語対応、詳細に記述することを推奨
  * 提出前にプラットフォームの禁止用語／セキュリティチェックを通過し、違反があった場合はエラーを直接返します
</ParamField>

<ParamField body="size" type="string" default="1:1">
  画面の比率

  公開用の比率値。システム内部では `resolution` によって特定のピクセル数に自動マッピングされます。

  13種類の比率をサポート：

  * `1:1` - 正方形構図（デフォルト、SNSのアイコン／ロゴ）
  * `3:2` - 横長構図（一眼レフカメラでよく使われる比率）
  * `2:3` - 縦長構図（ポスター縦型）
  * `4:3` - 横長構図（クラシックモニター／プレゼンテーション）
  * `3:4` - 縦長構図
  * `5:4` - 横長構図
  * `4:5` - 縦長構図（Instagram縦型投稿）
  * `16:9` - 横長構図（ワイドスクリーン動画のサムネイル）
  * `9:16` - 縦長構図（スマホ全画面／ショート動画のサムネイル）
  * `2:1` - 横長構図（ウェブバナー）
  * `1:2` - 縦長構図
  * `21:9` - 横長構図（映画の超ワイドスクリーン）
  * `9:21` - 縦長構図

  <Warning>
    比率の書き方のみ対応。ピクセル指定（例: `1024x1024`）は直接エラーになります
  </Warning>
</ParamField>

<ParamField body="resolution" type="string" default="1K">
  解像度の段階

  実際の画像の鮮明さを制御：

  * `1K` - 1024基準、コスト節約で日常使いに十分（デフォルト）
  * `2K` - 2048基準、ポスター／高画質ニーズ向け
  * `4K` - 3840基準、**6つの比率のみサポート**（`16:9` / `9:16` / `2:1` / `1:2` / `21:9` / `9:21`）

  <Warning>
    サポート外の4K組み合わせの場合は400エラーを返します：

    * `1:1` × `4K` ❌（ピクセル上限超過）
    * `3:2` / `2:3` × `4K` ❌（ピクセル上限超過）
    * `4:3` / `3:4` × `4K` ❌（ピクセル上限超過）
    * `5:4` / `4:5` × `4K` ❌（ピクセル上限超過）

    上記比率で高解像度が必要な場合は、`resolution=2K` を使用してください。
  </Warning>
</ParamField>

<ParamField body="quality" type="string" default="auto">
  画像の品質

  * `auto` - 自動（デフォルト、通常は `low` と同等）
  * `low` - 速くてコスト節約、輪郭は十分
  * `medium` - バランス型
  * `high` - 最高精度（4K + high は >120秒の処理時間）
</ParamField>

<ParamField body="image_urls" type="array">
  参考画像の配列。渡すと画像から画像生成モードになります

  * 最大 **9枚** の参考画像
  * 以下２つの形式をサポート：
    1. 公開アクセス可能なURL
    2. Base64エンコード形式
       * 完全なData URI形式である必要があります：`data:image/{フォーマット};base64,{base64データ}`
       * サポートされる画像フォーマット：jpeg、png、webp
       * 例：`data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABg...`
       * ⚠️ 注意：`data:image/jpeg;base64,` のプレフィックス部分が必須です
</ParamField>

## サイズ × 解像度マッピング表

`size × resolution` → OpenAI 実際のピクセル（13 比率 × 3 画質レベル）：

| size   | `1K`      | `2K`      | `4K`          |
| ------ | --------- | --------- | ------------- |
| `1:1`  | 1024×1024 | 2048×2048 | ❌ 超ピクセル上限       |
| `3:2`  | 1536×1024 | 2048×1360 | ❌ 超ピクセル上限       |
| `2:3`  | 1024×1536 | 1360×2048 | ❌ 超ピクセル上限       |
| `4:3`  | 1024×768  | 2048×1536 | ❌ 超ピクセル上限       |
| `3:4`  | 768×1024  | 1536×2048 | ❌ 超ピクセル上限       |
| `5:4`  | 1280×1024 | 2560×2048 | ❌ 超ピクセル上限       |
| `4:5`  | 1024×1280 | 2048×2560 | ❌ 超ピクセル上限       |
| `16:9` | 1536×864  | 2048×1152 | **3840×2160** |
| `9:16` | 864×1536  | 1152×2048 | **2160×3840** |
| `2:1`  | 2048×1024 | 2688×1344 | **3840×1920** |
| `1:2`  | 1024×2048 | 1344×2688 | **1920×3840** |
| `21:9` | 2016×864  | 2688×1152 | **3840×1648** |
| `9:21` | 864×2016  | 1152×2688 | **1648×3840** |

> `3:2` / `2:3` @ 2K は実際2048×1360（近似比率、誤差 \< 0.5%）；4K は6つの比率のみ対応（総ピクセルがOpenAI上限を超える組み合わせは使用不可）。

## タスク結果の照会

正常に送信されると `task_id` が返され、`GET /v1/tasks/{task_id}` を使用してタスクの状態をポーリングします。詳細は [タスク照会API](/ja/api-reference/tasks/get-task) を参照してください。

### ポーリングの推奨設定

* **初回照会の遅延**：送信後 10~20 秒待ってから照会を開始
* **照会間隔**：3~5 秒ごとを推奨
* **タイムアウトの参考値**：`high + 2K/4K` コンビネーションで最大約 130 秒かかるため、クライアントのタイムアウトは 180 秒以上を推奨
