## Body

<ParamField body="model" type="string" default="gpt-image-2-official" required>

  고정 입력 `gpt-image-2-official` (OpenAI 공식 gpt-image-2 모델)
</ParamField>

<ParamField body="prompt" type="string" required>
  이미지 생성을 위한 텍스트 설명

  * 한중영 지원, 상세 설명 권장
  * 제출 전 플랫폼의 민감 단어 / 보안 심사를 거침, 위반 내용 적발 시 즉시 오류 반환
</ParamField>

<ParamField body="size" type="string" default="1:1">
  화면 비율

  외부 사용 비율 값, 시스템 내부는 `resolution`에 따라 구체 픽셀로 자동 매핑

  13가지 비율 지원:

  * `1:1` - 정사각형 구성 (기본, SNS 프로필/로고)
  * `3:2` - 가로 구성 (DSLR 일반 비율)
  * `2:3` - 세로 구성 (포스터 세로판)
  * `4:3` - 가로 구성 (클래식 모니터 / PPT)
  * `3:4` - 세로 구성
  * `5:4` - 가로 구성
  * `4:5` - 세로 구성 (Instagram 세로 게시물)
  * `16:9` - 가로 구성 (와이드 스크린 비디오 커버)
  * `9:16` - 세로 구성 (휴대폰 전체 화면 / 짧은 동영상 커버)
  * `2:1` - 가로 구성 (웹 배너)
  * `1:2` - 세로 구성
  * `21:9` - 가로 구성 (영화 초광폭)
  * `9:21` - 세로 구성

  <Warning>
    비율 형태만 지원하며, 픽셀 치수(예: `1024x1024`) 입력 시 직접 오류가 발생합니다
  </Warning>
</ParamField>

<ParamField body="resolution" type="string" default="1K">
  해상도 레벨

  실제 이미지 선명도 제어:

  * `1K` - 1024 기준, 비용 절약 및 일상용 충분 (기본값)
  * `2K` - 2048 기준, 포스터 / 고화질 요구에 적합
  * `4K` - 3840 기준, **6가지 비율만 지원됨** (`16:9` / `9:16` / `2:1` / `1:2` / `21:9` / `9:21`)

  <Warning>
    지원하지 않는 4K 조합은 400 오류 반환:

    * `1:1` × `4K` ❌ (초과 픽셀 제한)
    * `3:2` / `2:3` × `4K` ❌ (초과 픽셀 제한)
    * `4:3` / `3:4` × `4K` ❌ (초과 픽셀 제한)
    * `5:4` / `4:5` × `4K` ❌ (초과 픽셀 제한)

    위 비율에서 고화질 필요 시, `resolution=2K` 사용 권장
  </Warning>
</ParamField>

<ParamField body="quality" type="string" default="auto">
  이미지 품질

  * `auto` - 자동 (기본, 일반적으로 `low`와 동일)
  * `low` - 빠른 처리 및 비용 절감, 윤곽선 중심
  * `medium` - 균형 잡힘
  * `high` - 최고 정밀도 (4K + high 시 소요 시간 > 120초)
</ParamField>

<ParamField body="image_urls" type="array">
  참조 이미지 배열, 입력 시 이미지 변환 모드 활성화

  * 최대 **9장** 참조 이미지 허용
  * 다음 두 가지 형식 지원:
    1. 공개 접근 가능한 URL
    2. Base64 인코딩 형식
       * 반드시 전체 Data URI 형식 사용: `data:image/{포맷};base64,{base64 데이터}`
       * 지원 이미지 포맷: jpeg, png, webp
       * 예시: `data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAYABg...`
       * ⚠️ 주의: `data:image/jpeg;base64,` 접두사 포함 필요
</ParamField>

## 크기 × 해상도 대응표

`size × resolution` → OpenAI 실제 픽셀 (13비율 × 3단계):

| size   | `1K`      | `2K`      | `4K`          |
| ------ | --------- | --------- | ------------- |
| `1:1`  | 1024×1024 | 2048×2048 | ❌ 초과 픽셀 한도       |
| `3:2`  | 1536×1024 | 2048×1360 | ❌ 초과 픽셀 한도       |
| `2:3`  | 1024×1536 | 1360×2048 | ❌ 초과 픽셀 한도       |
| `4:3`  | 1024×768  | 2048×1536 | ❌ 초과 픽셀 한도       |
| `3:4`  | 768×1024  | 1536×2048 | ❌ 초과 픽셀 한도       |
| `5:4`  | 1280×1024 | 2560×2048 | ❌ 초과 픽셀 한도       |
| `4:5`  | 1024×1280 | 2048×2560 | ❌ 초과 픽셀 한도       |
| `16:9` | 1536×864  | 2048×1152 | **3840×2160** |
| `9:16` | 864×1536  | 1152×2048 | **2160×3840** |
| `2:1`  | 2048×1024 | 2688×1344 | **3840×1920** |
| `1:2`  | 1024×2048 | 1344×2688 | **1920×3840** |
| `21:9` | 2016×864  | 2688×1152 | **3840×1648** |
| `9:21` | 864×2016  | 1152×2688 | **1648×3840** |

> `3:2` / `2:3` @ 2K는 실제로 2048×1360 (근사 비율, 오차 \< 0.5%)입니다; 4K는 6개 비율만 지원 (총 픽셀 수가 OpenAI 한도를 초과하는 조합은 사용할 수 없습니다).

## 작업 결과 조회

제출이 성공하면 `task_id`가 반환되며, `GET /v1/tasks/{task_id}` 를 통해 작업 상태를 폴링할 수 있습니다. 자세한 내용은 [작업 조회 엔드포인트](/kr/api-reference/tasks/get-task)를 참조하세요.

### 폴링 권장 사항

* **첫 조회 지연**: 제출 후 10~20초 기다린 후 조회 시작
* **조회 간격**: 3~5초 간격 권장
* **타임아웃 참고**: `high + 2K/4K` 조합은 최대 130초 소요될 수 있으며, 클라이언트 타임아웃은 180초 이상 권장
