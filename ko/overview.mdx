APIMart는 이미지 및 비디오 생성 서비스를 제공하는 통합 AI 콘텐츠 생성 API 플랫폼입니다. 간단한 REST API 호출을 통해 GPT-4o, Gemini, Sora2, VEO3 등 다양한 주류 AI 모델을 사용하여 고품질 이미지 및 비디오 콘텐츠를 생성할 수 있습니다.

## API 워크플로

### 1. 생성 작업 제출

모든 이미지 및 비디오 생성은 비동기 처리 방식을 채택합니다. 작업을 제출하면 시스템은 즉시 `task_id`를 반환합니다:

```bash
curl --request POST \
  --url http://pxz-api.xinsulv.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "一只可爱的猫咪在花园里",
      "aspect_ratio": "1:1"
    }
  }'
```

### 2. 작업 상태 조회

반환된 `task_id`를 사용하여 작업 진행 상황과 결과를 조회합니다:

```bash
curl --request POST \
  --url http://pxz-api.xinsulv.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'
```

### 3. 생성 결과 받기

작업이 완료되면 응답에 생성된 이미지 또는 비디오 URL이 포함됩니다:

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## 핵심 기능

### 비동기 작업 처리

모든 생성 요청은 비동기 처리 방식을 채택하여 긴 대기 시간을 방지합니다:

- `task_id` 즉시 반환
- 작업 상태 API를 통한 진행 상황 조회
- webhook 콜백 알림 지원 (선택 사항)
- 작업 상태 실시간 업데이트: `pending` → `processing` → `completed`

### 다중 모델 지원

여러 이미지 및 비디오 생성 모델을 지원하는 통합 API 엔드포인트:

- **이미지 생성**: GPT-4o, Gemini, Seedream 등 5개 모델
- **비디오 생성**: Sora2, VEO3 등 4개 모델
- 통일된 요청 형식 및 응답 구조
- 유연한 매개변수 구성

### 투명한 가격 책정

크레딧 기반 과금, 투명하고 명확함:

- 각 작업에서 예상 소비 크레딧 반환
- 작업 완료 후 실제 소비 크레딧 표시
- 성공적으로 생성된 콘텐츠에만 비용 청구
- 숨겨진 비용 없음

### 유연한 구성

다양한 생성 매개변수 구성 지원:

- **이미지**: 여러 종횡비 (1:1, 3:2, 2:3)
- **비디오**: 여러 종횡비 (16:9, 9:16), 재생 시간 (10초, 15초), 해상도
- 참조 이미지 지원 (이미지에서 이미지, 이미지에서 비디오)
- 사용자 지정 메타데이터

## 인증 방법

모든 API 요청에는 Bearer Token 인증이 필요합니다:

```
Authorization: Bearer YOUR_API_KEY
```

  <Accordion title="이미지와 비디오 생성에는 얼마나 걸리나요?">
    모든 생성 작업은 비동기로 처리됩니다. 이미지 생성은 일반적으로 몇 초에서 수십 초가 걸리고, 비디오 생성은 재생 시간과 해상도에 따라 일반적으로 1-5분이 걸립니다. 작업을 제출하면 즉시 task_id가 반환되며, 상태 조회 API를 통해 진행 상황을 추적할 수 있습니다.
  </Accordion>

  <Accordion title="작업 상태를 어떻게 조회하나요?">
    작업 제출 시 반환된 `task_id`를 사용하여 작업 상태 조회 API를 사용합니다:

    ```bash
    POST http://pxz-api.xinsulv.com/api/openapi/query
    ```

    자세한 내용은 [작업 상태 조회 문서](/ko/api-reference/tasks/status)를 참조하세요.
  </Accordion>

  <Accordion title="webhook 콜백을 지원하나요?">
    네, 작업 제출 시 `callback_url` 매개변수를 전달할 수 있습니다. 작업이 완료되면 시스템이 해당 주소로 완전한 작업 결과를 포함한 POST 요청을 보냅니다.
  </Accordion>

  <Accordion title="과금은 어떻게 이루어지나요?">
    APIMart는 크레딧 기반 과금 시스템을 채택합니다:

    - 작업 제출 시 예상 소비 크레딧(`estimated_credits`) 반환
    - 작업 완료 후 실제 소비 크레딧(`credits_used`) 반환
    - 성공적으로 생성된 콘텐츠에만 비용 청구, 실패한 작업은 청구되지 않음
    - 모델에 따라 크레딧 소비량이 다름
  </Accordion>

  <Accordion title="어떤 이미지 및 비디오 형식을 지원하나요?">
    - **이미지**: JPEG, PNG 등 일반적인 형식 지원
    - **비디오**: MP4 형식으로 출력
    - **참조 이미지**: 공개적으로 액세스 가능한 이미지 URL 입력 지원
  </Accordion>
</AccordionGroup>
