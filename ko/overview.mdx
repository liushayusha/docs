## API 작업 흐름

### 1. 생성 작업 제출

모든 이미지 및 비디오 생성은 비동기 처리 방식으로 진행됩니다. 작업을 제출할 때 `out_task_id`(발신자 작업 ID)를 전달해야 하며, 시스템은 `task_id`와 `out_task_id`를 반환합니다:

```bash
curl --request POST \
  --url https://aireiter.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "꽃밭에 있는 귀여운 고양이",
      "aspect_ratio": "1:1"
    },
    "out_task_id": "my_task_123456"
  }'
```

### 2. 작업 상태 조회

반환된 `task_id` 또는 제출 시 사용한 `out_task_id` 중 하나로 작업 진행 상황과 결과를 조회합니다:

```bash
# task_id 사용 조회
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "task_id": "order_xxxxxxxxxxxx"
  }'

# 또는 out_task_id 사용 조회
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "out_task_id": "my_task_123456"
  }'
```

### 3. 생성 결과 가져오기

작업이 완료되면 응답에 생성된 이미지 또는 비디오 URL이 포함됩니다:

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "task_id": "order_xxxxxxxxxxxx",
    "out_task_id": "my_task_123456",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## 핵심 기능

### 비동기 작업 처리

모든 생성 요청은 비동기 처리 방식을 사용하여 장시간 대기를 방지합니다:

- 작업 제출 시 `out_task_id`(발신자 작업 ID, 비즈니스 식별용)를 전달해야 합니다
- 즉시 시스템에서 생성한 `task_id`와 제출한 `out_task_id`를 반환합니다
- `task_id` 또는 `out_task_id`로 작업 상태를 조회할 수 있습니다(둘 중 하나)
- webhook 콜백 알림 지원(선택 사항)
- 작업 상태 실시간 업데이트: `pending` → `processing` → `completed`

### 다중 모델 지원

통합 API 엔드포인트로 다양한 이미지 및 비디오 생성 모델을 지원합니다:

- **이미지 생성**: GPT-4o, Gemini, Seedream 등 5개 모델
- **비디오 생성**: Sora2, VEO3 등 4개 모델
- 통합된 요청 형식과 응답 구조
- 유연한 파라미터 설정

### 투명한 가격 정책

포인트 기반 과금으로 투명하고 명확합니다:

- 각 작업에 대해 예상 소모 포인트를 반환
- 작업 완료 후 실제 소모 포인트 표시
- 성공적으로 생성된 콘텐츠에 대해서만 비용 부과
- 숨겨진 비용 없음

### 유연한 설정

다양한 생성 파라미터 구성을 지원합니다:

- **이미지**: 다양한 가로세로 비율(1:1, 3:2, 2:3)
- **비디오**: 다양한 가로세로 비율(16:9, 9:16), 길이(10초, 15초), 해상도
- 참조 이미지 지원(이미지에서 이미지, 이미지에서 비디오)

  </Accordion>

  <Accordion title="out_task_id 는 무엇인가요?">
    `out_task_id` 는 발신자 작업 ID 로, 작업 제출 시 직접 설정하는 값입니다:

    - **필수 파라미터**: 작업 제출 시 반드시 `out_task_id` 를 전달해야 합니다
    - **비즈니스 식별자**: 시스템 내 주문번호나 비즈니스 ID 등을 사용할 수 있습니다
    - **유연한 조회**: 시스템에서 반환된 `task_id` 를 저장하지 않아도 `out_task_id` 로 직접 작업 상태를 조회할 수 있습니다
    - **결과 연동**: 조회 결과에 `task_id` 와 `out_task_id` 가 함께 반환되어 비즈니스 데이터 연동이 용이합니다
  </Accordion>

  <Accordion title="과금 방식은 어떻게 되나요?">
    AIReiter 는 포인트 제도로 과금합니다:

    - 작업 제출 시 예상 소모 포인트(`estimated_credits`) 가 반환됩니다
    - 작업 완료 후 실제 소모 포인트(`credits_used`) 가 반환됩니다
    - 성공적으로 생성된 콘텐츠에 대해서만 비용이 청구되며, 실패한 작업은 비용이 발생하지 않습니다
    - 모델마다 포인트 소모량이 다릅니다
  </Accordion>

  <Accordion title="지원하는 이미지 및 비디오 형식은 무엇인가요?">

    - **비디오**: MP4 형식으로 출력됩니다
    - **참고 이미지**: 공개 접근 가능한 이미지 URL 입력을 지원합니다
