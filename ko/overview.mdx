## API 작업 흐름

### 1. 생성 작업 제출

모든 이미지 및 비디오 생성은 비동기 처리 모드를 사용합니다. 작업을 제출할 때 `out_task_id`(발신자 작업 ID)를 전달해야 하며, 시스템은 `out_task_id`를 반환합니다:

```bash
curl --request POST \
  --url https://aireiter.com/api/openapi/submit \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "gpt_4o",
    "params": {
      "prompt": "한 귀여운 고양이가 정원에 있습니다",
      "aspect_ratio": "1:1"
    },
    "out_task_id": "my_task_123456"
  }'
```

### 2. 작업 상태 조회

제출한 `out_task_id`를 사용하여 작업 진행 상황과 결과를 조회하세요:

```bash
curl --request POST \
  --url https://aireiter.com/api/openapi/query \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "out_task_id": "my_task_123456"
  }'
```

### 3. 생성 결과 가져오기

작업이 완료되면 응답에 생성된 이미지 또는 비디오 URL이 포함됩니다:

```json
{
  "statusCode": 200,
  "message": "",
  "data": {
    "out_task_id": "my_task_123456",
    "status": "completed",
    "output": [
      {
        "url": "https://s1.pxz.ai/upload/image-generator/xxx.jpeg"
      }
    ]
  }
}
```

## 핵심 기능

### 비동기 작업 처리

모든 생성 요청은 비동기 처리 모드를 사용하여 긴 대기 시간을 피합니다:

- 작업 제출 시 `out_task_id` (발신자 작업 ID, 비즈니스 식별용) 전달 필요
- 제출한 `out_task_id` 즉시 반환
- `out_task_id`를 사용하여 작업 상태 조회 가능
- webhook 콜백 알림 지원(선택 사항)
- 작업 상태 실시간 업데이트: `pending` → `processing` → `completed`

### 다중 모델 지원

하나의 통합 API 엔드포인트에서 다양한 이미지 및 비디오 생성 모델 지원:

- **이미지 생성**: GPT-4o, Gemini, Seedream 등 5개 모델
- **비디오 생성**: Sora2, VEO3 등 4개 모델
- 통합된 요청 포맷 및 응답 구조
- 유연한 파라미터 구성

### 투명한 요금 정책

포인트 기준 과금으로 투명하고 명확함:

- 각 작업에 대해 예상 소모 포인트 반환
- 작업 완료 후 실제 소모 포인트 표시
- 성공적으로 생성된 콘텐츠에 대해서만 요금 부과
- 숨겨진 비용 없음

### 유연한 구성

다양한 생성 파라미터 설정 지원:

- **이미지**: 다양한 가로 세로 비율 (1:1, 3:2, 2:3)
- **비디오**: 다양한 가로 세로 비율 (16:9, 9:16), 길이 (10초, 15초), 해상도
- 참조 이미지 지원 (이미지에서 이미지 생성, 이미지에서 비디오 생성)

  <Accordion title="out_task_id란 무엇인가요?">
    `out_task_id`는 작업을 요청하는 쪽에서 설정하는 작업 ID입니다:

    - **필수 파라미터**: 작업 제출 시 반드시 `out_task_id`를 전달해야 합니다
    - **업무 식별**: 시스템 내 주문 번호, 업무 ID 등으로 식별 가능
    - **유연한 조회**: `out_task_id`로 직접 작업 상태 조회 가능
    - **결과 연동**: 조회 결과에 `out_task_id`가 포함되어 업무 데이터와 연동하기 용이함
  </Accordion>

  <Accordion title="요금은 어떻게 청구되나요?">
    AIReiter는 포인트제로 청구됩니다:

    - 작업 제출 시 예상 소모 포인트(`estimated_credits`) 반환
    - 작업 완료 후 실제 소모 포인트(`credits_used`) 반환
    - 성공적으로 생성된 내용에 대해서만 요금 청구, 실패한 작업은 청구되지 않음
    - 모델에 따라 포인트 소모량 상이
  </Accordion>

  <Accordion title="지원하는 이미지 및 비디오 형식은 무엇인가요?">
    - **이미지**: JPEG, PNG 등 일반적인 형식 지원
    - **비디오**: MP4 형식 출력
    - **참고 이미지**: 공개 접근 가능한 이미지 URL 입력 지원
  </Accordion>
